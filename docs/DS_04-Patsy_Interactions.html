patsy

Bringing Statsmodels (R) formula notation to sklearn !

…and automatically get:

  • Design matrices with dummy coding
  • Polynomial terms, interactions, splines, contrasts
  • Feature Names
  • A record of transformations, so predictions on new data work
  • No manual preprocessing (no OneHotEncoder, no PolynomialFeatures)

All of this is extremely useful before passing the data into sklearn, which only expects numeric matrices.

import pandas as pd
import patsy
from sklearn.linear_model import LinearRegression

# ---------------------------
# 1. Create example data
# ---------------------------
df = pd.DataFrame({
    "y":   [1, 2, 3, 4, 5, 6],
    "x1":  [1, 2, 3, 4, 5, 6],
    "x2":  [2, 1, 2, 1, 2, 1],
    "grp": ["A", "B", "A", "B", "A", "B"]
})

# ---------------------------
# 2. Build the design matrices with patsy
# ---------------------------
# R-style formula:
formula = "y ~ x1 + x2 + C(grp) + x1:x2 + I(x1**2)"

formula = "y ~ x1*x2 + C(grp) + I(x1**2)"

y, X = patsy.dmatrices(formula, df, return_type="dataframe")

print(X)

9 Predicting on New Data

To avoid mismatched dummy columns, reuse the same formula + design info!

Your friend: patsy.build_design_matrices()

new = pd.DataFrame({
    "x1": [10],
    "x2": [2],
    "grp": ["A"]
})
# build X_new based on the *training* formula & coding
X_new = patsy.build_design_matrices([X.design_info], new)[0]

patsy handles:

  • the missing C(grp)[T.B] column
  • dummy coding
  • correct interaction term

No feature engineering mistakes.

patsy gives us:

→ A declarative way to specify transformations

→ Consistency between training and prediction

→ Readability (“the design matrix is the formula made numeric”)

→ Less boilerplate compared to sklearn pipelines for simple models
